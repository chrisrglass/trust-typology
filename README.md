# The Trust Typology v2 — Local Prototype

Five classes × twelve dimensions in six blocks (Fidelity / Fairness / Contribution / Respect / Honesty / Accountability), built on the v22 instrument.
**Research prototype: provisional, pre-fielding.** All prevalences, item
probabilities, and class content derive from the synthetic design study, not
from survey respondents. Class names are working labels under a naming embargo.

## Run it

```bash
npm install    # once
npm run dev    # dev server (default http://localhost:5173)
# or
npm run build && npm run preview   # production build at http://localhost:4173
```

## Verify the classifier

```bash
npm run verify   # 100 synthetic respondents + 5 archetypes; expect ~76% / 5-of-5
```

## What this mirrors

Architecture and design system are ported from the original site
(`../trust-typology`, the live seven-community typology — untouched by this
prototype): Vite + React 19, hash routing, one global `App.css`, lucide icons,
CSS-bar charts. Pages: quiz flow (`#/`), About (`#/typology`), Explore Types
(`#/profiles`, `#/profiles/:id`), Dimensions (`#/dimensions`, `#/dimensions/:id`).

## What changed from the original

- **Typology:** 5 classes (Institutional Reformers, Results-Oriented
  Pragmatists, Populist Insurgents, Fair-Deal Skeptics,
  Grassroots Communitarians), each with a theory of reform.
- **Dimensions:** 12 in six blocks (Fidelity / Fairness / Contribution / Respect / Honesty / Accountability), grouped by observed co-variation;
  dimension cards grouped by layer.
- **Quiz:** the full v22 form (24 paired items + G01 + covariates). Classification
  is naive-Bayes profile matching on the 24 items only; G01 is reflected back as
  "your theory of reform" and never used for classification (v22 coding rule 2).
  No backend: responses stay in the browser.
- **Provenance data:** `src/data/classifierParams.js` and `scripts/fixture_n100.json`
  are auto-generated by `claude-code/scratch/trust-v21-robustness/export_site_data_v2.py`.

## Before any public deploy (not done, on purpose)

Set a real `base` in `vite.config.js`; restore per-type social cards; replace the
ShareBar URL; verify all proofPoint URLs; decide Supabase persistence; and, above
all: field the instrument — the typology should not ship publicly as findings
until real respondents produce it.
